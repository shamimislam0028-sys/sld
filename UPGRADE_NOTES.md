# UPGRADE — যেকোনো Word ফাইল → AI দিয়ে JSON MCQ (v201.2)

## যা হলো
DOCX ইম্পোর্ট এখন **Google Gemini (free)** দিয়ে যেকোনো ফরম্যাটের MCQ বের করে।
ফ্লো: `docx → টেক্সট বের করা → Gemini → পরিষ্কার JSON → pptx জেনারেটর`।

- **Primary:** প্রতিটা ফাইল Gemini দিয়ে পার্স হয় (সবচেয়ে নির্ভরযোগ্য)।
- **Safety-net:** key না থাকলে বা ফ্রি কোটা শেষ হলে অ্যাপ ভাঙে না — তখন বিল্ট-ইন
  রোবাস্ট রেগেক্স-পার্সারে নিজে থেকে ফিরে যায়।

বদলানো/নতুন ফাইল:
- `api/gemini-extract.js`  ← নতুন, শেয়ার্ড Gemini মডিউল
- `api/parse-docx.js`      ← Gemini primary + regex fallback (Vercel এটাই সার্ভ করে)
- `backend/server.js`      ← একই লজিক (লোকাল ডেভ)

কোনো নতুন npm ডিপেন্ডেন্সি লাগে না (Node-এর বিল্ট-ইন `fetch` ব্যবহার করে)।

## ★ যা করতেই হবে: Gemini API key বসানো

1. https://aistudio.google.com/apikey  → **Create API key** (ক্রেডিট কার্ড লাগে না)।
2. **Vercel-এ:** প্রজেক্ট → Settings → Environment Variables → যোগ করুন:
   - `GEMINI_API_KEY` = আপনার key
   - (ঐচ্ছিক) `GEMINI_MODEL` = `gemini-2.5-flash`  (Google মডেল রিনেম করলে এখানে বদলাবেন)
   → Save → **Redeploy**।
3. **লোকাল টেস্ট:** রুটে একটা `.env` ফাইলে `GEMINI_API_KEY=...` রাখুন, তারপর `npm run dev`।

key না বসালেও অ্যাপ চলবে — শুধু রেগেক্স-পার্সারে চলবে (আগের মতো)।

## ফ্রি টিয়ার (মাঝ-২০২৬)
- Gemini Flash ফ্রি টিয়ার: দিনে ~১,৫০০ রিকোয়েস্ট, কনটেক্সট ১০ লাখ টোকেন — একটা গোটা
  প্রশ্নব্যাংক এক কলেই যায়। মিনিটে ~১৫ রিকোয়েস্টের লিমিট আছে।
- খুব বড় ফাইলে (১০০+ প্রশ্ন) আউটপুট লিমিটে আটকালে ফাইলটা ভাগ করে দিন।
- ফ্রি টিয়ারে Google প্রম্পট ট্রেনিংয়ে ব্যবহার করতে পারে — MCQ পার্সিংয়ে সাধারণত সমস্যা নয়।

## রেসপন্স
`/api/parse-docx` আগের মতোই `{ mcqs: [...] }` দেয় (সাথে debug-এর জন্য `source` ফিল্ড:
`gemini` / `regex-fallback` / `regex-no-key`)। ফ্রন্টএন্ডে কিছু বদলাতে হয়নি।

## ডিপ্লয়
zip-এ `node_modules` নেই — Vercel নিজে ইনস্টল করে। ফোল্ডার push/import → env var বসান → deploy।

## v201.3 — দুটো ফিক্স

**১) ম্যাথ ঠিকমতো রেন্ডার:** api-এর টেক্সট এক্সট্রাক্টর এখন OMML → **LaTeX (`$$...$$`)** করে
(`api/omml-to-latex.js`, backend থেকে port করা)। আগে `1/2x` জাতীয় লসি হিন্ট যেত; এখন
`$$\frac{1}{2x}$$` যায়, যা pptx ইঞ্জিন সরাসরি রেন্ডার করে। Gemini-কেও এই LaTeX দেওয়া হয়
আর প্রম্পটে বলা আছে হুবহু রাখতে।

**২) শেয়ার্ড উদ্দীপক সব প্রশ্নে:** directive-এ ("৪ ও ৫ নম্বর") ক'টা প্রশ্ন আছে গুনে passage
এখন ওই সবগুলো প্রশ্নেই বসে (আগে শুধু প্রথমটায় যেত)। regex + Gemini দুই পাথেই ঠিক করা।

নতুন ফাইল: `api/omml-to-latex.js` (শেয়ার্ড)।

## v201.4 — চারটে আপডেট

**১) ম্যাট্রিক্স:** `ommlToLatex`-এ OMML matrix (`<m:m>/<m:mr>/<m:e>`) → LaTeX
`\begin{matrix}...\end{matrix}` যোগ করা (api ও backend দুটোতেই)। এখন ম্যাট্রিক্স
আসল 2D ম্যাট্রিক্স হয়ে রেন্ডার হয়, সংখ্যা জোড়া লাগে না।

**২ ও ৩) Auto-fit (shrink-to-fit):** প্রশ্ন ও অপশন বক্সে PowerPoint-এর
`<a:normAutofit/>` বসানো (`pptEngine.js`-এর `setShapeText`)। প্রশ্ন বা অপশন বড় হলে
টেক্সট নিজে ছোট হয়ে fixed বক্সে বসে — উপচে পড়ে না। (বক্স রিসাইজ/সরানোর বদলে shrink-to-fit
বেছে নেওয়া হয়েছে, কারণ গ্রুপ-করা শেপ/ব্যাকগ্রাউন্ডের সাথে সেটা নিরাপদ ও পরিষ্কার।)

**৪) Model fallback:** Gemini এখন একটা মডেল ফেল করলে (rate-limit/error) পরেরটায় যায়।
ডিফল্ট চেইন: `gemini-2.5-flash → gemini-2.5-flash-lite → gemini-3-flash-preview`
(আপনার ss.png-এর ফ্রি-টিয়ার মডেলগুলোর সাথে মিলিয়ে)। env দিয়ে বদলানো যায়:
`GEMINI_MODELS="modelA,modelB,modelC"` (কমা দিয়ে, ক্রম অনুযায়ী চেষ্টা করবে)।
শুধু ফেল করলেই সুইচ করে — সফল হলে ওখানেই থামে।

## v201.5 — matrix ও autofit ঠিক করা

**Matrix (আসল কারণ):** regex পাথ ঠিক ছিল, কিন্তু **Gemini পাথে** LLM ম্যাট্রিক্সের রো-সেপারেটর
`\\` নষ্ট করছিল (JSON-এ ব্যাকস্ল্যাশ), তাই ৩ রো একলাইনে মিশে "52","62" হচ্ছিল।
সমাধান: Gemini-কে পাঠানোর আগে প্রতিটা `$$...$$` ব্লক একটা টোকেন (〖M0〗) দিয়ে ঢেকে
দেওয়া হয় — LLM ম্যাথে হাতই দেয় না — ফেরত এলে আসল LaTeX বসিয়ে দেওয়া হয়।
(api/gemini-extract.js: protectMath/restoreMath)

**Autofit (আসল কারণ):** `normAutofit`-এ কোনো `fontScale` ছিল না, তাই রেন্ডারার টেক্সট
ছোট করত না — বড় প্রশ্ন (৪০pt) উপচে পড়ত। বক্সগুলো গ্রুপ-করা বলে জিওমেট্রি সরাসরি পড়া
যায় না, তাই fixed বক্সের জন্য ক্যালিব্রেটেড **length-based fontScale** বসানো হয়েছে
(ম্যাথ ব্লককে কম্প্যাক্ট ধরে)। এখন লম্বা প্রশ্ন/অপশন explicit fontScale নিয়ে ছোট হয়ে বক্সে
বসে; ছোট কনটেন্ট অপরিবর্তিত থাকে। run-লেভেল শ্রিংক বন্ধ (getScaleFactor→1)।
